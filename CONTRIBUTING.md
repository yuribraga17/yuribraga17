# 🤝 Contribuindo

Obrigado pelo interesse em contribuir com os meus projetos! Toda ajuda é
bem-vinda — de correções de bugs a novas ideias.

## 🐞 Reportando bugs

1. Verifique se já não existe uma [issue](../../issues) sobre o assunto.
2. Abra uma nova issue descrevendo:
   - O que aconteceu e o que era esperado
   - Passos para reproduzir
   - Logs, prints ou mensagens de erro, se houver
   - Ambiente (SO, versão, etc.)

## ✨ Sugerindo melhorias

Abra uma issue explicando **o problema que a sugestão resolve** e a solução
proposta. Contexto ajuda a priorizar.

## 🔀 Enviando código

1. Faça um _fork_ do repositório.
2. Crie uma branch descritiva:
   ```bash
   git checkout -b feat/minha-feature
   # ou
   git checkout -b fix/descricao-do-bug
   ```
3. Faça commits objetivos seguindo o padrão **Conventional Commits**:
   ```
   feat(escopo): adiciona X
   fix(escopo): corrige Y
   ```
4. Abra um Pull Request descrevendo a mudança e o motivo.

## 🧭 Boas práticas

- Mantenha o código limpo, legível e consistente com o estilo do projeto.
- Comentários e textos ao usuário em **pt-BR**.
- Teste localmente antes de enviar.
- Não exponha segredos, tokens ou credenciais.
- Prefira mudanças pequenas e focadas.
